# 言序语言 · VS Code

言序编程语言的官方 VS Code 扩展。

当前扩展版本为`1.4.0`，面向言序核心`1.1.7`及兼容的后续 1.x 版本。扩展会索引格式 2 包工作区、递归导入的外部模块和当前 CLI 的 25 个标准模块，并为言窗等锁定包的公开成员提供签名补全、导航与拼音检索。离线标准库快照记录对应核心版本，避免插件与运行时 API 静默漂移。

## 功能

- `.yx` 文卷语法高亮；
- 中文引号、括号自动配对与代码折叠；
- 块结构、分支、错误处理、协议和文档注释的智能缩进；
- 变量、流程、函数、类、父类调用、类型判断、协议、泛型、异步任务、网络、模块、元组和切片代码片段；
- 编辑器右上角一键运行当前文卷；
- 1.0 LSP 实时诊断、补全、跳转、引用、重命名、悬停、文档符号与全文格式化；
- 索引类、协议、字段、方法、函数、参数、变量、常量和导入别名，未保存的编辑会增量更新；
- 递归索引`引「相对路径」`和`引「包:依赖别名/公开模块」`引用的工作区内外模块；格式 2 包导入遵守根依赖、显式导出和传递依赖隔离，仅向使用方暴露模块的`公`有声明；
- 从`yanxu 标准库 --json`读取当前安装版本的标准库模块与成员，命令不可用时使用随扩展发布的快照；
- 在代码补全和“转到工作区中的符号”中支持中文、无声调全拼和拼音首字母，例如`jisuanzonghe`或`jszh`可找到`计算总和`；
- 为索引符号提供成员补全、悬停、参数提示和跨文卷定义跳转，标准库定义可在只读虚拟文卷中查看；
- LSP 不可用时仍提供基础补全、悬停说明和内建函数签名提示；
- VS Code“运行和调试”中的源码断点、继续、单步、调用栈与作用域变量；
- 静态检查、字节码 VM、执行踪迹、工作区测试、YXB 编译与桌面应用 Bundle 命令；
- 一键调用 `yanxu 迁 --写` 更新当前文卷中的弃用语法；
- 在可复用 REPL 中运行选区或当前行；
- 状态栏显示语言服务状态，并可一键重启或查看通信日志；
- 自动发现“运行、检查、VM、踪迹、测试、编译、Bundle、迁移”八类 VS Code 任务；
- 一键调用`yanxu 迁 --写`迁移当前文卷中的已弃用形式；
- 命令面板打开言序 REPL 或在线文档。

## 使用

请先[安装言序 1.1.7 命令行](https://yanxu.dev/download/)，然后打开任意 `.yx` 文件或包含`言序.toml`的项目。扩展自身直接连接核心 CLI，因此需要可用的`yanxu`。默认会从 `PATH` 查找；若安装在其他位置，请设置 `yanxu.executablePath`。

打开命令面板并输入“言序”即可查看所有命令。编辑器右键菜单可以运行当前文卷或把选中代码送入 REPL；“终端 → 运行任务”中可以直接选择言序任务。“构建发布版桌面应用 Bundle”会调用`yanxu compile <工作区> --release --bundle`。输入“言窗应用”可插入带窗口、布局和退出回调的完整骨架。打开`.yx`后可在行号左侧设置断点，再从“运行和调试”选择“调试当前言序文卷”。

## 设置

| 设置 | 默认值 | 用途 |
| --- | --- | --- |
| `yanxu.executablePath` | `yanxu` | 指定言序命令行程序路径 |
| `yanxu.saveBeforeRun` | `true` | 执行、检查或跟踪前保存文卷 |
| `yanxu.languageServer.enabled` | `true` | 自动启动实时诊断和格式化服务 |
| `yanxu.index.enabled` | `true` | 建立工作区、导入模块和标准库符号索引 |
| `yanxu.index.maxFiles` | `5000` | 限制一次索引扫描的`.yx`文卷数量 |
| `yanxu.index.includeExternalModules` | `true` | 递归索引工作区外的本地与包导入模块 |
| `yanxu.completion.pinyin` | `true` | 使用全拼或拼音首字母筛选补全与工作区符号 |
| `yanxuLanguageServer.trace.server` | `off` | 控制 LSP 通信日志详细程度 |

索引会随文卷创建、修改、保存和删除自动更新。依赖或外部模块大规模变化后，可从命令面板运行“言序：重建符号索引”。

## 自动发布

每次提交或 Pull Request 都会运行结构校验、单元测试和 VSIX 打包。更新`package.json`与`package-lock.json`版本、加入同版本变更记录并推送到`main`后，`Auto Release`会自动创建`v*`标签，再由可复用`Release`工作流创建 GitHub Release 并上传 VSIX；仓库设置了`VSCE_PAT` Secret 时，还会同步发布到 VS Code Marketplace。已有标签也可从 Actions 手工重跑发布。
